## DFS
$\texttt{Deepth First Search}$

核心思想就是：

> 不撞南墙不回头

“回头”就是 `return`。

框架：
```cpp
int ans[N],vis[N];
void dfs(int dep){
    if(边界条件) return;
    进行某种操作
    for(遍历邻居){
        if(vis[i]) continue;
        vis[i] = ans[dep] = i;
        dfs(i);
        vis[i] = ans[dep] = 0;
    }
}
```
### P1706 全排列
__Problem__  
给定 $n$，输出从 $1$ 到 $n$ 的数字的全排列，场宽为 $5$。

__Solve__  
首先要输入 $n$：
```cpp
int n; cin >> n;
```

然后，考虑算法。  
这里因为 $n$ 不确定，所以不能使用枚举，即考虑搜索，`dfs(int cnt)`，其中，`cnt` 为当前枚举的数字个数，如果 $cnt > n$ 就进行输出。  
所以现在考虑输出函数 `print`，这样就要使用数组 `ans` 来存储答案，方便输出，也就是从 $ans_1$ 输出到 $ans_n$ 即可，保留场宽可以用 $\texttt{setw(5)}$ 或 $\texttt{printf("%5d",ans[i])}$  
通过手画递归树可知，对于之前已经用过的数字不应该使用，所以要用一个数组 `vis` 来记录是否被用，并且每次都要从 $1$ 开始搜。  
```text
例如 dfs(3,1)
dfs(1,1)->dfs(2,2)[vis:1,1,0]->dfs(3,3)[vis:1,1,1]->dfs(4,4)[vis:1,1,1]->return cout << "1     2     3" << endl;
                  [vis:1,0,0]<---------[vis:1,1,0]<-+
        +>dfs(3,2)[vis:1,0,1]->dfs(2,3)[vis:1,1,1](这就是为什么要从1开始)
...(剩下的自己脑补吧)
```

在每次搜索完后都要回溯操作，否则那个数字再其他搜索树分支就**不会被使用**。

所以，注意事项：
1) 记得保存答案
2) 记得标记访问
3) 记得回溯重置

```cpp
#include <iostream>
#include <iomanip>//使用 setw 要包含头文件
using namespace std;
const int N = 11;
int n,ans[N],vis[N];
void print(){
    for(int i = 1;i <= n;i++) cout << setw(5) << ans[i];
    cout << endl;//细节
}
void dfs(int cnt){
    if(cnt > n){
        print();
        return;//告辞
    }
    for(int i = 1;i <= n;i++){//每次都从 1 开始查找有哪些没有使用过
        if(vis[i]) continue;//如果使用过就不再用了
        vis[i] = ans[cnt] = i;//一定要记录答案 并标记
        dfs(cnt+1);//传送个数
        vis[i] = ans[cnt] = 0;//回溯 这个ans不重置也行，后面会覆盖掉
    }
}
int main(){
    cin >> n;
    dfs(1);//从第一个开始遍历 因为第一个cnt穿进去必须是1，ans数组才能从1开始记录
    return 0;
}
```

你不会真以为这就行了吧？一测试连样例都过不了。

观察结果发现，有顺序逆过来的，原来组合是考虑顺序的，而不是像上一题一样的排列，所以要仔细读题。

所以我们应该用一个参数来记录当前的位置，每次都往后找，不回头：
```cpp
#include <iostream>
#include <iomanip>
using namespace std;
const int N = 24;
int n,r;
int ans[N],vis[N];
void print(){
    for(int i = 1;i <= r;i++) cout << setw(3) << ans[i];
    cout << endl;
}
void dfs(int now,int cnt){
    if(cnt > r){
        print();//超出边界
        return;
    }
    for(int i = now;i <= n;i++){
        if(vis[i]) continue;//同上一题
        vis[i] = ans[cnt] = i;
        dfs(i,cnt+1);
        vis[i] = ans[cnt] = 0;
    }
}
int main(){
    cin >> n >> r;
    dfs(1,1);
    return 0;
}
```
### P1219 八皇后
__Problem__  
[自己看](https://www.luogu.com.cn/problem/P1219)

__Solve__

首先判断算法，因为不知棋盘大小，所有用搜索。

而且在这道题中，需要模拟前半部分，就会明白：
- 在搜索过程中，会有答案被修改的情况
- 需要判断三个方面是否有皇后
- 也需要回溯
- 要多输出一个方案数


我们可以**逐行**搜索，最后到第 $n$ 行的就 `return`，并用 `sun` 记录方案数，如果 `sum <= 3` 就输出，否则一直到最后才输出 `sum`，那么就要用 `ans` 保存答案用于输出，即：
```cpp
int n,sum;
void dfs(int line){
    if(line > n){
        sum++;
        if(sum > 3) return;//如果当前方案数已经大于 3 了，就退出
        for(int i = 1;i <= n;i++) cout << ans[i] << " ";//n 行 n 列，所以输出到 ans[n]
        cout << endl;//记得换行
        return;//这也要退出 别忘了
    }
    for(int i = 1;i <= n;i++){//从当前行的第一个位置开始往后遍历计算方案
        //do something to solution
    }
}
```
会发现，可以准备三个数组存放**每一行是否有棋子**的 `row`、**每一左对角线（/）是否有棋子**的 `lef`、**每一有对角线（\）是否有棋子**的 `rig`。

把行和列合并是因为只要当前行有棋子，就不用考虑列了。

****
**进行模拟，总结规律**

::::info[归纳过程]
(这里只进行规律总结的演示，对于题意的模拟自己画)

对于 $row$，只用在每一行标记即可，$row_i$ 为 $0$ 才放置    
对于 $lef$，如果一个一个的写一定会 $MLE$，所以要压缩一下：  
$\    $发现，处于 $(1,1)$ 所在左对角线上的点为 $(1,1)$                          他(们)的行数 $i$ 加上列数 $j$ 都等于一个定值 $2$
$\    $发现，处于 $(1,2)$ 所在左对角线上的点为 (1,2) (2,1)                      他们的行数   $i$ 加上列数 $j$ 都等于一个定值 $3$  
$\    $发现，处于 $(1,3)$ 所在左对角线上的点为 (1,3) (2,2) (3,1)……              他们的行数   $i$ 加上列数 $j$ 都等于一个定值 $4$
$\    $所以，处于 $(i,j)$ 所在左对角线上是否有棋子，只用判断 `lef[i+j]` 是否为 $1$，如果为 $0$ 才放置。
对于 $rig$，和 $lef$ 一样都需要压缩：  
$\    $发现：处于 $(1,1)$ 所在右对角线上的点为 $(1,1)\ (2,2)\ (3,3)\ (4,4)$……他们的列数 $j$ 减去行数 $i$ 都等于一个定值 $0$  
$\    $发现：处于 $(1,2)$ 所在右对角线上的点为 $(1,2)\ (2,3)\ (3,4)\ (4,5)$……他们的列数 $j$ 减去行数 $i$ 都等于一个定值 $1$ 
$\    $发现：处于 $(5,4)$ 所在右对角线上的点为 $(2,1)\ (3,2)\ (4,3)\ (5,4)$……他们的列数 $j$ 减去行数 $i$ 都等于一个定值 $2$
$\    $……
$\    $所以，处于 $(i,j)$ 所在右对角线上是否有棋子，只用判断 `rig[j-i]` 是否为 $1$，如果为 $0$ 才放置。  
$\    $但是这个 $j-i$ 可能会为**负数**，最小为 $1-n$，所以我们可以再加上一个定值 $n$，结果还是定值但是避免了负数的存在。  
$\    $最后整理一下：rig[j-i+n]/rig[i-j+n]
    
这样就可以把在**同一行、同一左对角线、同一右对角线**上是否有棋子的情况都表示出来了，空间复杂度为 $O(n)$，这样查询也方便。
::::
*****
当然我们也可以合并成一个数组来检查，如 `vis[N][3]`，起到一个顶三个的作用。

这样代码就简单了：
```cpp
#include <iostream>
using namespace std;
const int N = 30;
int n,sum,vis[N][3],ans[N];
void dfs(int line){//逐行搜索
    if(line > n){
        sum++;
        if(sum > 3) return;//不输出
        for(int i = 1;i <= n;i++) cout << ans[i] << " ";
        cout << endl;//换行
        return;//记得退出
    }
    for(int i = 1;i <= n;i++){//从当前行的第一个位置往又走，i 相当于 j
        if(vis[i][0] || vis[i+line][1] || vis[i-line+n][2]) continue;//使用过
        vis[i][0] = vis[i+line][1] = vis[i-line+n][2] = 1;
        ans[line] = i;//记录当前行的答案 如果有不合适会被修改 因为我们有回溯
        dfs(line+1);//下一行
        vis[i][0] = vis[i+line][1] = vis[i-line+n][2] = ans[line] = 0;//回溯
    }
}
int main(){
    cin >> n;
    dfs(1);
    cout << sum << endl;//记得输出
    return 0;
}
```
